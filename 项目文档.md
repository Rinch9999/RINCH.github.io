# Person of Interest 粉丝网站 - 项目文档

## 目录结构

网站采用清晰的文件分层结构，遵循前端开发最佳实践：

```
/github-websnet/
├── index.html                    # 首页
├── characters_timeline/          # 角色时间线目录
│   ├── characters_timeline.html  # 角色时间线总览页面
├── timeline_summary.html         # 时间线页面
├── season/                       # 剧集页面目录
│   ├── episodes.html             # 剧集列表页面
│   ├── season1.html              # 第一季页面
│   ├── season2.html              # 第二季页面
│   ├── season3.html              # 第三季页面
│   ├── season4.html              # 第四季页面
│   ├── season5.html              # 第五季页面
│   └── episodes-shared.css       # 剧集共享样式
├── characters_timeline/          # 角色时间线目录
│   ├── finch_timeline.html       # 芬奇时间线
│   ├── reese_timeline.html       # 里瑟时间线
│   ├── root_timeline.html        # 根时间线
│   ├── shaw_timeline.html        # 肖时间线
│   └── characters_timeline.css   # 角色时间线样式
├── characters/                   # 角色页面目录
│   ├── finch.html                # 芬奇角色页面
│   ├── reese.html                # 里瑟角色页面
│   ├── root.html                 # 根角色页面
│   ├── shaw.html                 # 肖角色页面
│   └── characters.css            # 角色共享样式
├── js/                           # JavaScript脚本目录
├── styles/                       # CSS样式目录
└── images/                       # 图片资源目录
```

## 页面结构说明

### 1. 首页 (index.html)
- 顶部导航栏：包含logo、菜单项和移动端汉堡菜单
- 主轮播区：展示剧集精彩剧照和简介
- 角色展示区：以卡片形式展示主要角色
- 剧集亮点：突出展示剧集特色和成就
- 页脚：包含版权信息和相关链接

### 2. 角色页面 (characters.html)
- 角色列表：以网格布局展示所有角色卡片
- 角色详情：点击卡片展开或跳转到详细信息
- 搜索功能：提供角色筛选和搜索

### 3. 时间线页面 (timeline_summary.html)
- 按年份组织的事件列表
- 左右分列式时间轴布局
- 详细的事件描述和关键信息

## 样式架构

### 1. 样式文件组织
- `base.css`: 基础样式，包括重置、全局变量和通用类
- `components.css`: 组件样式，如按钮、卡片、表单等
- `responsive.css`: 响应式媒体查询，针对不同设备的样式调整
- `main.css`: 主样式文件，整合其他样式并包含特定页面样式

### 2. 设计规范
- 颜色方案：深色主题为主，采用剧集相关的配色
- 字体：无衬线字体，保证良好可读性
- 间距：统一的间距规范，使用CSS变量定义
- 阴影和动画：适度使用，提升用户体验但不过度

## JavaScript功能模块

### 1. 主脚本 (main.js)
- 页面初始化
- 导航栏交互
- 通用工具函数
- 图片懒加载实现
- 无障碍功能支持

### 2. 轮播组件 (carousel.js)
- 自动轮播功能
- 手动控制和指示器
- 响应式调整

### 3. 角色页面脚本 (characters.js)
- 角色卡片交互
- 搜索和筛选逻辑
- 详情展示功能

## 无障碍实现细节

### 1. ARIA标签使用
- 为所有非语义元素添加适当的角色属性
- 使用aria-labelledby和aria-describedby建立关联
- 为交互元素添加适当的aria状态

### 2. 键盘导航
- 确保所有交互元素可通过键盘访问
- 提供清晰的焦点样式
- 支持Tab键导航和回车键激活

### 3. 辅助功能支持
- 文本与背景的足够对比度
- 支持系统高对比度模式
- 支持减少动画偏好设置

## 性能优化策略

### 1. 图片优化
- 实现懒加载，减少初始加载时间
- 使用适当的图片格式和压缩
- 预加载关键图片资源

### 2. 资源管理
- 合并和压缩CSS文件
- 延迟加载非关键JavaScript
- 利用浏览器缓存

### 3. 渲染优化
- 使用CSS变量减少重复代码
- 避免布局抖动
- 优化CSS选择器性能

## 响应式设计实现

### 1. 断点设置
- 移动设备：360px - 767px
- 平板设备：768px - 1199px
- 桌面设备：1200px及以上

### 2. 适配策略
- 流式布局和弹性盒模型
- 图片响应式调整
- 导航栏在移动设备上转为汉堡菜单
- 网格布局在小屏幕上调整为单列
